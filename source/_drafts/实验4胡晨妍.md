### 1、实验目的

本次实验旨在使用静态站点构建工具（Hexo）构建一个静态Web博客站点，了解和掌握Web应用开发的基本要素与流程（环境准备→项目初始化→内容编写→主题配置→本地预览→部署上线），理解静态站点生成器（SSG）的工作原理及其与动态站点的区别。

### 2、实验环境

- 操作系统：Windows 11 专业版 22H2
- 软件版本：Node.js v20.10.0，npm v10.2.3，Git 2.43.0，Hexo CLI 4.3.0
- 部署平台：GitHub Pages
- 博客主题：NexT v8.19.0
- 文本编辑器：Visual Studio Code 1.85.0
- 网络：互联网连接（需访问GitHub）

### 3、实验过程

#### 步骤一：验证环境与安装Hexo CLI

确认Node.js、npm和Git已正确安装：

```bash
node -v    # v20.10.0
npm -v     # 10.2.3
git --version  # git version 2.43.0
```

全局安装Hexo CLI：

```bash
npm install -g hexo-cli
hexo -v    # 验证安装
```

![图1：环境依赖版本验证](4.步骤一.png)

#### 步骤二：初始化博客项目

```bash
cd D:\Blog
hexo init my-blog
cd my-blog
npm install
```

初始化完成后，项目目录结构如下：

```
my-blog/
  ├── _config.yml          # 站点配置文件
  ├── package.json         # 项目依赖清单
  ├── scaffolds/           # 文章模板
  ├── source/              # 内容源文件目录
  │   └── _posts/          # 文章存放目录
  └── themes/              # 主题目录
```

![图2：Hexo博客项目目录结构](./4.步骤二.PNG)

#### 步骤三：本地预览与创建文章

```bash
hexo server                # 启动本地预览
```

访问 http://localhost:4000 查看默认博客。创建文章并生成：

```bash
hexo new "Web站点构建实验"
hexo generate              # 生成静态页面
```

![图3：创建的新文章在博客首页显示](./4.步骤三.png)

#### 步骤四：更换NexT主题

```bash
git clone https://github.com/theme-next/hexo-theme-next themes/next
```

修改 `_config.yml` 中的 `theme` 字段为 `next`，重新生成并预览。

![图4：应用NexT主题后的博客首页](./4.步骤五.png)

#### 步骤五：部署到GitHub Pages

创建GitHub仓库（命名为 `用户名.github.io`），安装部署插件并配置：

```bash
npm install hexo-deployer-git --save
```

在 `_config.yml` 末尾添加：

```yaml
deploy:
  type: git
  repo: https://github.com/你的用户名/你的用户名.github.io.git
  branch: main
```

执行部署：

```bash
hexo clean && hexo generate && hexo deploy
```

访问 `https://你的用户名.github.io` 查看上线博客。

![图5：部署到GitHub Pages后的线上博客](./4.步骤四.PNG)

### 5、知识总结

**静态站点生成器（SSG）工作原理：**

```
Markdown源文件 + 主题模板 + 配置文件
        ↓
   [Hexo构建]
        ↓
  纯HTML/CSS/JS静态文件
        ↓
   [部署到Web服务器]
        ↓
      用户访问
```

**静态站点 vs 动态站点：**

| 对比维度 | 静态站点（SSG） | 动态站点（如WordPress） |
|----------|----------------|------------------------|
| 页面生成时机 | 构建时预先生成所有页面 | 用户访问时由服务器实时生成 |
| 数据库 | 不需要 | 通常需要 |
| 访问速度 | 极快（纯静态文件） | 较慢（需执行后端代码+查数据库） |
| 安全性 | 高（无后端攻击面） | 需额外防护 |
| 部署方式 | 上传静态文件即可 | 需配置服务器环境和数据库 |
| 适用场景 | 博客、文档站、个人网站 | 电商、社交、CMS |

**Web应用开发流程（本实验覆盖）：**

环境准备 → 项目初始化 → 内容编写 → 主题配置 → 本地预览 → 部署上线

### 6、出现问题

| 问题现象 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 'hexo'不是内部或外部命令 | Hexo CLI未正确安装 | 执行 `npm install -g hexo-cli` 重新安装 |
| hexo init速度极慢或失败 | 网络问题，无法访问GitHub | 配置代理，或使用Gitee镜像源 |
| hexo server启动后浏览器无法访问 | 端口4000被占用 | 更换端口：`hexo server -p 4001` |
| 部署后GitHub Pages显示404 | 仓库名称不是 `用户名.github.io` | 检查仓库命名是否正确 |
| 部署后页面样式丢失 | 站点URL配置错误 | 检查 `_config.yml` 中 `url` 字段设置为 `https://用户名.github.io` |
