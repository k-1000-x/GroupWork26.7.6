### 1、实验目的

本次实验旨在安装Visual Studio Code（VS Code）文本代码编辑器，安装和配置Markdown相关插件，掌握代码编辑器插件的基本概念，学习Markdown基础语法，并使用VS Code完成Markdown文档的编辑和渲染任务。

### 2、实验环境

-操作系统：Windows 11 专业版 22H2
-软件版本：Visual Studio Code 1.85.0
-VS Code插件：Markdown All in One v3.6.0、Markdown Preview Enhanced v0.8.0、Paste Image v1.0.4、markdownlint v0.55.0
-网络：互联网连接（用于下载VS Code及插件）

### 3、实验过程

#### 步骤一：安装Visual Studio Code

访问VS Code官网下载安装包，双击启动安装向导。在"选择附加任务"页面，勾选以下选项：
-创建桌面快捷方式
-将"通过Code打开"操作添加到Windows资源管理器文件/目录上下文菜单
-将Code注册为受支持的文件类型的编辑器
-添加到PATH（必须勾选）

点击"安装"完成安装。

![图1：VS Code安装时界面](./3.步骤一.png)

#### 步骤二：安装Markdown相关插件

点击左侧活动栏中的扩展图标（或按 `Ctrl+Shift+X`），依次搜索并安装以下插件：

1. **Markdown All in One** — 提供目录生成、快捷键支持、自动完成、列表自动延续等功能
2. **Markdown Preview Enhanced** — 提供数学公式（LaTeX）、流程图（Mermaid）、导出为PDF/HTML等功能
3. **Paste Image** — 支持直接粘贴剪贴板中的图片到Markdown文件
4. **markdownlint** — 检查Markdown语法错误和格式规范

![图2：已安装的Markdown相关插件列表](./3.步骤二.png)

#### 步骤三：配置编辑器优化写作体验

按 `Ctrl+,` 打开设置，进行以下配置：
-`files.autosave` → 设置为 `afterDelay`（开启自动保存）
-`editor.wordwrap` → 设置为 `on`（开启单词折行）
-Paste Image插件：设置 `Path Type` 为 `relative`

![图3：VS Code自动保存和单词折行设置](./3.步骤三.png)

#### 步骤四：创建并编辑Markdown文件

新建文件（`Ctrl+N`），保存为 `test.md`，输入以下内容：

```markdown
# 我的第一个Markdown文档

## 什么是Markdown？

Markdown是一种**轻量级**的标记语言。

## Markdown的核心特点

-**简洁性**：仅需少量符号即可完成排版
-**可读性**：纯文本仍然清晰易读
-**兼容性**：支持GitHub、博客平台等多种场景

## 一个代码示例

```javascript
function hello() {
    console.log("Hello, Markdown!");
}
```

使用 `Ctrl+Shift+V` 打开预览窗口，实时查看渲染效果。

### 5、知识总结

**代码编辑器 vs 集成开发环境（IDE）：**

| 类型 | 特点 | 典型代表 |
|------|------|----------|
| 代码编辑器 | 轻量级、启动快、以编辑为核心功能、通过插件扩展能力 | VS Code、Sublime Text、Notepad++ |
| 集成开发环境（IDE） | 功能全面、集成了编译/调试/构建/版本控制等全套工具链 | IntelliJ IDEA、PyCharm、Eclipse |

**Markdown核心语法：**

| 语法元素 | Markdown写法 | 渲染效果 |
|----------|--------------|----------|
| 一级标题 | `# 标题` | <h1>标题</h1> |
| 二级标题 | `## 标题` | <h2>标题</h2> |
| 加粗 | `**文本**` | **文本** |
| 列表 | `- 项目` | • 项目 |
| 代码块 | \```language ... \``` | 带语法高亮的代码块 |
| 链接 | `[文本](URL)` | 可点击链接 |
| 图片 | `![alt](路径)` | 嵌入图片 |

### 6、出现问题

| 问题现象 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 安装VS Code后无法在命令行使用code命令 | 安装时未勾选"添加到PATH" | 重新运行安装程序勾选"添加到PATH"；或手动将VS Code安装目录添加到系统环境变量 |
| 扩展面板搜索不到插件 | 网络问题或搜索关键词不准确 | 检查网络连接；使用准确的插件全名搜索 |
| `[TOC]`不生成目录 | 未安装Markdown All in One插件 | 安装并启用Markdown All in One插件 |
| 预览中图片无法显示 | 图片路径错误 | 检查图片路径是否正确，建议使用相对路径 |

### 7、心得体会

通过本次实验，我深刻体会到"工欲善其事，必先利其器"的道理。VS Code作为一款轻量级但功能强大的代码编辑器，其插件生态极为丰富。Markdown All in One和Markdown Preview Enhanced等插件将编辑器从单纯的文本编辑工具扩展为高效的写作平台。Markdown的简洁语法让我能够专注于内容本身而非排版细节，这对于技术写作来说尤为重要。同时，我也认识到插件管理是现代软件开发工具的核心能力之一——通过插件组合，一个基础工具可以适应各种不同的应用场景。
