
## 1. 实验目的
安装VS Code代码编辑器，熟悉软件界面布局；安装Markdown系列扩展插件；掌握编辑器自定义配置；学习Markdown基础语法，完成文档编写与实时预览。

## 2. 实验环境
- 操作系统：Windows 11
- 软件：Visual Studio Code
- 插件：Markdown All in One、Markdown Preview Enhanced、Paste Image

## 3. 实验过程
1. 官网下载VS Code安装包，安装时勾选添加PATH、右键文件打开菜单等附加功能；
2. 启动软件，熟悉活动栏、侧边栏、编辑区、底部面板、状态栏布局；
![侧边栏](/images/2.1.png)
1. 快捷键`Ctrl+Shift+X`打开扩展面板，安装Markdown配套插件；
![安装Markdown配套插件](/images/2.2.png)
1. 设置自动保存、自动单词折行，配置粘贴图片相对路径；
![设置](/images/2.3.png)
1. 新建`.md`文件，编写标题、列表、代码块、引用等Markdown语法，打开预览窗口查看渲染效果。
![渲染效果](/images/2.4.png)

## 4. 实验结果
VS Code正常启动，所有插件成功启用；Markdown文档渲染正常，目录、代码块、图片均可正常展示。

## 5. 知识总结
1. 代码编辑器VS IDE：VS Code轻量、启动快，依靠插件拓展功能；IDE集成编译、调试全套工具；
2. Markdown：轻量级标记语言，语法简洁，可转换HTML、PDF，广泛用于文档、博客；
3. 插件机制：VS Code核心扩展方式，不同插件分别实现目录生成、流程图绘制、文件导出等功能；
4. 相对路径可规避预览图片失效问题，是Markdown写作推荐路径格式。

## 6. 出现问题
1. 终端无法识别`code`命令
    - 原因：安装未勾选添加到系统PATH
    - 解决：重装VS Code并勾选PATH选项
2. Markdown预览窗口空白、加载异常
    - 原因：插件未启用、软件后台进程冲突
    - 解决：启用对应插件，完全重启VS Code
3. Markdown内图片无法显示
    - 原因：使用绝对路径存放图片
    - 解决：修改为相对路径存放图片

## 7. 心得体会
VS Code强大的插件生态使其适配各类开发场景，轻量化特性对比大型IDE启动速度更快。Markdown语法简单易上手，是技术文档、笔记、博客的标准写作格式。编辑器预览类故障大多由插件、残留进程导致，遇到异常优先重启软件、检查插件状态，可快速解决大部分渲染问题。
