## 1、实验目的

本次实验旨在掌握 Linux 系统下用户账户的创建、查看、修改和删除操作；理解用户组的概念，掌握组的管理方法；理解 Linux 文件权限模型（读、写、执行），掌握使用 `chmod` 和 `chown` 命令修改文件权限和所有者。

## 2、实验环境

| 项目 | 内容 |
|------|------|
| 操作系统 | Windows 11 专业版 |
| 虚拟机软件 | Oracle VirtualBox 7.2.12 |
| 虚拟机系统 | Ubuntu Server 22.04.5 LTS |
| 实验节点 | node1（192.168.56.102） |
| 连接方式 | SSH 远程登录（端口 2223） |
| 初始用户 | vboxuser（具有 sudo 权限） |

## 3、实验过程

### 步骤一：用户管理——创建新用户

通过 SSH 连接到 node1，使用 `adduser` 命令创建名为 `testuser` 的新用户：

```bash
sudo adduser testuser
```

执行过程中，系统提示设置密码（设置为 `123456`），其他信息（全名、房间号等）直接按回车跳过，最后输入 `Y` 确认。

![创建新用户](/images/5.1.jpg)

### 步骤二：用户管理——查看用户信息

查看 `/etc/passwd` 文件，确认用户已成功创建：

```bash
cat /etc/passwd | grep testuser
```

输出结果：

```
testuser:x:1001:1001:..:/home/testuser:/bin/bash
```

`/etc/passwd` 各字段含义：

| 字段 | 说明 |
|------|------|
| `testuser` | 用户名 |
| `x` | 密码占位符（实际密码存储在 `/etc/shadow`） |
| `1001` | UID（用户 ID） |
| `1001` | GID（主组 ID） |
| `/home/testuser` | 用户主目录 |
| `/bin/bash` | 登录 Shell |

### 步骤三：用户管理——授予 sudo 权限

将 `testuser` 添加到 `sudo` 组，使其拥有执行管理员命令的权限：

```bash
sudo usermod -aG sudo testuser
```

查看用户所属组：

```bash
groups testuser
```

输出：

```
testuser : testuser sudo
```

### 步骤四：用户管理——切换用户

切换到 `testuser`，验证用户是否可正常登录：

```bash
su - testuser
```

输入密码后，提示符变为 `testuser@node1:~$`，说明切换成功。

![切换用户验证](/images/5.2.jpg)

返回 `vboxuser`：

```bash
exit
```

### 步骤五：组管理——创建组

创建名为 `devgroup` 的新组，并将 `vboxuser` 加入该组：

```bash
sudo groupadd devgroup
sudo usermod -aG devgroup vboxuser
groups vboxuser
```

输出：

```
vboxuser : vboxuser adm cdrom sudo dip plugdev lxd devgroup
```

![组管理操作](/images/5.3.jpg)

### 步骤六：文件权限管理——创建并查看文件

创建测试文件并查看初始权限：

```bash
touch test.txt
ls -l test.txt
```

输出：

```
-rw-rw-r-- 1 vboxuser vboxuser 0 Jul 11 11:51 test.txt
```

权限解读：`-rw-rw-r--` 表示这是一个普通文件，所有者可读可写，组内用户可读可写，其他用户只读。

### 步骤七：文件权限管理——数字法修改权限

使用数字法将文件权限修改为 `755`：

```bash
chmod 755 test.txt
ls -l test.txt
```

输出：

```
-rwxr-xr-x 1 vboxuser vboxuser 0 Jul 11 11:51 test.txt
```

**权限数字对照表：**

| 数字 | 权限 | 含义 |
|------|------|------|
| 7 | rwx | 读 + 写 + 执行 |
| 6 | rw- | 读 + 写 |
| 5 | r-x | 读 + 执行 |
| 4 | r-- | 只读 |
| 0 | --- | 无权限 |

`755` 表示：所有者 `7`（rwx 可读可写可执行），组 `5`（r-x 可读可执行），其他人 `5`（r-x 可读可执行）。

### 步骤八：文件权限管理——修改所有者

将文件所有者修改为 `root`：

```bash
sudo chown root test.txt
ls -l test.txt
```

输出：

```
-rwxr-xr-x 1 root vboxuser 0 Jul 11 11:51 test.txt
```

文件所有者从 `vboxuser` 变为 `root`。

![修改文件所有者](/images/5.4.jpg)

## 5、知识总结

### 5.1 用户相关命令

| 命令 | 作用 |
|------|------|
| `sudo adduser 用户名` | 创建新用户 |
| `cat /etc/passwd` | 查看所有用户信息 |
| `sudo usermod -aG 组名 用户名` | 将用户添加到组 |
| `groups 用户名` | 查看用户所属组 |
| `su - 用户名` | 切换用户 |
| `sudo deluser 用户名` | 删除用户 |

![用户命令汇总](/images/5.8.jpg)

### 5.2 组相关命令

| 命令 | 作用 |
|------|------|
| `sudo groupadd 组名` | 创建新组 |
| `cat /etc/group` | 查看所有组信息 |
| `sudo groupdel 组名` | 删除组 |

![组命令汇总](/images/5.9.jpg)

### 5.3 权限管理命令

| 命令 | 作用 |
|------|------|
| `chmod 755 文件名` | 数字法修改权限 |
| `chmod u+x 文件名` | 符号法：所有者增加执行权限 |
| `chmod g-w 文件名` | 符号法：组去掉写权限 |
| `sudo chown 用户 文件名` | 修改文件所有者 |
| `sudo chgrp 组名 文件名` | 修改文件所属组 |
| `ls -l 文件名` | 查看文件权限 |

![权限命令汇总](/images/5.10.jpg)

## 7、心得体会

通过本次实验，我深入学习了 Linux 用户管理、组管理和文件权限管理，掌握了 `adduser`、`usermod`、`chmod`、`chown` 等核心命令的使用方法。通过实际操作创建用户、配置权限，我对 Linux 多用户系统的权限模型有了更直观的理解。
