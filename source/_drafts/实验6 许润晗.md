# Linux综合实验报告

### 一、实验目的

1. 掌握登录后查看用户、系统内核、系统版本、主机名、工作目录等基础信息命令。

2. 熟练使用命令完成目录切换、文件/文件夹创建、复制、移动、删除等文件系统操作。

3. 学会查看内存、磁盘、进程、目录占用大小等系统资源状态。

4. 掌握文件内容查看、输出重定向、管道、通配符的基础使用。

5. 学会使用apt工具完成软件源更新、软件安装与卸载操作。

6. 读懂/etc/passwd、/etc/shadow、/etc/group三大用户配置文件，理解用户字段含义。

7. 熟练完成用户创建、设置密码、赋予管理员权限、切换用户、删除用户全套操作。

8. 熟记Linux权限数字对照表，掌握数字法、符号法两种chmod权限修改方式。

9. 使用chown修改文件所有者与所属组，chgrp单独修改文件所属用户组。

10. 区分普通用户与root管理员权限，理解sudo临时提权机制。

11. 掌握环境变量查看、临时环境变量创建与删除。

12. 区分用户级永久环境变量与系统级全局环境变量，掌握三种系统配置文件写法。

13. 理解各配置文件加载时机、生效范围，学会source命令即时刷新配置。

14. 掌握自定义命令别名alias配置。

15. 熟悉环境配置常见报错排查方法。

## 三、实验操作步骤

### 第一部分：实验四 Linux基本操作

步骤一：登录与查看系统信息

登录Ubuntu终端，依次执行如下命令查看系统基础信息：
whoami               # 查看当前登录用户
uname -a             # 查看内核与硬件架构完整信息
lsb_release -a       # 查看Ubuntu系统版本信息
hostname             # 查看本机主机名
pwd                  # 查看当前所在工作目录

步骤二：文件与目录的基本操作

完成文件夹、文件创建、复制、移动、删除等操作：
ls -lh                    # 列出目录内容，展示详细权限、人性化文件大小
cd ~                      # 切换到用户家主目录
mkdir test_dir            # 创建文件夹test_dir
touch test.txt            # 创建空白文本文件test.txt
cp test.txt test_copy.txt # 复制test.txt为test_copy.txt
mv test.txt /tmp/         # 将test.txt移动到/tmp临时目录
rm -f test.txt            # 强制删除文件test.txt

步骤三：查看系统资源状态

监控内存、磁盘、进程、文件夹占用资源：
free -h           # 人性化单位查看内存使用情况
df -h             # 查看磁盘分区整体占用
top               # 实时查看系统进程、CPU与内存占用，按q退出界面
du -sh *          # 查看当前目录下各个子目录总占用大小
![](./6.3.jpg)
步骤四：文件内容查看、管道与重定向

1. 文件读取查看命令
cat /etc/hosts                  # 一次性完整输出文件内容
less /etc/netplan/*.yaml        # 分页浏览配置文件，上下翻页查看
head /var/log/auth.log          # 查看日志文件前10行
tail /var/log/auth.log          # 查看日志文件末尾10行
tail -f /var/log/auth.log       # 实时追踪日志新增内容
2. 输出重定向
ls -lh > filelist.txt   # 将目录列表覆盖写入filelist.txt
pwd >> filelist.txt     # 将当前路径追加写入文件，不覆盖原有内容
3. 管道过滤操作
ip a | grep inet        # 查看网卡信息，筛选出IP地址行
ls -l /etc | less       # 目录列表分页查看
4. 通配符批量匹配文件
ls *.txt    # 匹配所有后缀为txt的文件
ls test*    # 匹配所有以test开头的文件、文件夹
步骤五：apt软件包管理
sudo apt update         # 更新本地软件源索引缓存
sudo apt install tree   # 安装tree目录树形查看工具
sudo apt remove tree    # 卸载已安装的tree软件

步骤八：文件权限管理

权限数字对照表
数字 权限 含义 
7 rwx 读+写+执行 
6 rw- 读+写 
5 r-x 读+执行 
4 r-- 只读 
0 --- 无任何权限 

权限修改与归属操作命令
chmod 755 test.txt                     # 数字法修改文件权限
chmod u+x test.txt                    # 符号法：给文件所有者增加执行权限
chmod g-w test.txt                    # 符号法：移除同组用户的写权限
sudo chown root:root test.txt         # 修改文件所有者为root，所属组为root
sudo chgrp www-data test.txt          # 仅修改文件所属用户组为www-data
### 第三部分：实验六 Linux环境配置

步骤九：查看环境变量
env                  # 查看系统全部环境变量
echo $PATH           # 查看PATH可执行文件搜索路径
echo $HOME           # 查看当前用户家目录
echo $USER           # 查看当前登录用户名
![](./6.14.jpg)
步骤十：临时设置环境变量

仅当前终端会话有效，关闭终端后失效。
**追加自定义目录到PATH**
export PATH=$PATH:/home/dev/mybin
**创建自定义变量**
export MY_VAR="Hello World"
**验证变量**
echo $MY_VAR
**除自定义临时变量**
unset MY_VAR
![](./6.15.jpg)
步骤十一：永久设置环境变量（用户级，仅当前用户生效）
配置文件 加载时机 作用范围 
~/.bashrc 每次打开新终端时执行 当前用户 
~/.profile 用户登录时执行一次 当前用户 
~/.bash_profile 用户登录时执行（优先级高于.profile） 当前用户 

编辑用户配置文件：
nano ~/.bashrc
文件末尾添加内容：
**扩展程序搜索路径**
export PATH=$PATH:/home/dev/mybin
**自定义环境变量**
export MY_CUSTOM_VAR="自定义变量"
**自定义命令别名**
alias ll='ls -alF'
保存退出，立即生效：
source ~/.bashrc
![](./6.16.jpg)
步骤十二：永久设置环境变量（系统级，所有用户生效）

方法1：/etc/environment
修改后需重启系统才生效，无需export，直接赋值字符串：
sudo nano /etc/environment
写入内容：
PATH="/usr/local/sbin:...:/my/custom/path"
方法2：/etc/profile
所有用户登录时自动加载，支持export，修改后可手动刷新：
sudo nano /etc/profile
文件末尾添加：
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
export PATH=$PATH:$JAVA_HOME/bin
立即全局生效
source /etc/profile
方法3（推荐）：/etc/profile.d/
新建独立配置脚本，便于维护，系统自动读取该目录所有.sh文件：
sudo nano /etc/profile.d/myenv.sh
写入全局变量
export MY_GLOBAL_VAR="全局变量"
刷新生效
source /etc/profile.d/myenv.sh

### 四、实验过程遇到的问题与解决方案

1. 问题：使用rm删除文件夹提示“目录不为空”
原因：rm命令默认仅支持删除文件，无法直接删除文件夹。
解决：使用递归强制删除命令 rm -rf 文件夹名。

1. 问题：执行apt install提示权限不足
原因：系统软件安装修改系统文件，普通用户无操作权限。
解决：命令前添加sudo，临时获取管理员权限执行。
