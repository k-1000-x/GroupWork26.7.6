## 实验七 远程登录管理
### 1. 实验目的
安装、配置OpenSSH远程登录服务；配置SSH密钥免密登录；使用SCP、rsync完成跨服务器文件传输与增量同步。

### 2. 实验环境
两台Ubuntu Server虚拟机，Host-Only内网网段`192.168.56.0/24`

### 3. 实验过程
1. 安装`openssh-server`，设置SSH服务开机自启，修改`sshd_config`禁止root远程登录；
2. 使用`ssh-keygen`生成Ed25519公私密钥对，`ssh-copy-id`推送公钥至远程服务器；

![ssh-copy-id](/images/7.2.png)
3. 测试免密SSH远程登录；
![远程登录](/images/7.3.png)
4. 使用`scp`上传、下载单文件与文件夹；安装rsync工具，实现跨服务器增量文件同步。

### 4. 实验结果
SSH服务正常运行，支持密码、密钥两种登录方式；免密登录配置完成；SCP、rsync均可稳定传输、同步服务器文件。

### 5. 知识总结
1. SSH：加密远程登录协议，默认22端口，保障公网、内网传输数据安全；
2. 两种认证方式对比：
    - 密码认证：操作简单，安全性较低，适合临时测试；
    - 密钥认证：公私钥配对验证，安全性高，生产环境推荐使用；
3. 文件传输工具：
    - SCP：简易一次性文件复制，适合少量文件传输；
    - rsync：增量同步，仅传输修改文件，适合定期备份、目录同步。

### 6. 出现问题
1. SSH连接提示`Connection refused`
    - 原因：远程服务器SSH服务未启动
    - 解决：`sudo systemctl start ssh`启动服务并设置开机自启
2. 配置密钥后登录仍需要输入密码
    - 原因：`~/.ssh/authorized_keys`文件权限过高
    - 解决：执行`chmod 600 ~/.ssh/authorized_keys`修改权限
3. 远程服务器提示`rsync: command not found`
    - 原因：目标服务器未安装rsync工具
    - 解决：远程执行`sudo apt install rsync -y`安装工具

### 7. 心得体会
SSH是远程管理Linux服务器的标准工具，密钥登录相比密码登录安全系数更高，正式服务器应当关闭密码登录功能。rsync增量同步相比SCP传输效率更高，适合项目、日志长期备份。多服务器运维场景下，掌握远程文件同步工具可以极大减少重复文件传输操作，提升运维效率。
